## Princípios de Formatação
A formatação do código é essencial para a legibilidade e manutenção.

Um código bem formatado facilita a compreensão e a colaboração entre os membros da equipe.

A formatação correta reduz o tempo gasto para entender o código e diminui a probabilidade de introduzir erros.

#### Formatação Vertical
A formatação vertical se refere a maneira como o código é distribuído ao longo da página. É importante que o código esteja organizado de forma a facilitar a leitura de cima para baixo. Isso inclui a organização de classes, métodos e blocos de códigos de uma maneira lógica e coerente.

#### Formatação Horizontal
A formatação horizontal diz respeito ao uso de espaços e alinhamentos dentro de uma linha de código. Um bom uso de espaços pode tornar o código mais legível e menos confuso. Por exemplo, espaços ao redos de operadores e após vírgulas em listas de parâmetros ajudam a melhorar a clareza.

#### Regras de Equipe
Cada equipe deve definir e seguir um conjunto de regras de formatação para garantir a consistência em todo o código-base. Isso inclui convenções sobre indentação, espaçamento, nomes de variáveism entre outros. Seguir essas regras é crucial para facilitar a revisão de código e a colaboração.

#### Regras de formatação do Uncle Bob
Uncle Bob, em seu livro "Clean Code", sugere várias práticas de formatação que contribuem para a clareza e manutenção do código. Entre elas estão o uso consistente de indentação, linhas em branco para separar blocos de código relacionados, e nomes de variáveis e métodos claros e descritivos.

## Abstração e Encapsulamento
#### Estruturas de Dados
A programação orientada a objetos é um pilar essencial no desenvolvimento de software moderno.
Entender e aplicar corretamente os conceitos de objetos e estruturas de daods é fundamental para criar sistemas robustos, escaláveis e manuteníveis.
Objetos e estruturas de dados formam a base para o gerenciamento eficiente de dados e a implementação de lógica de negócios.
Utilizando as melhores práticas de design, conseguimos garantir a integridade dos dados, facilitar a colaboração em equipe e promover a reutilização de código.
